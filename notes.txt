

Идея проекта:
Сендбокс-шутер в стиле Alien Shooter и Alien Swarm
Основные свойства:
- Проект на Unity3D
- Вид от третьего лица сверху
- Управление WASD+мышь
- Кооператив/PvP (используя Photon Network или другой подобный инструмент. Возможно было бы круто найти p2p решение, что бы не нужно было платить большие деньги если игроков будет много)
- Наличие режиме кооперативного sandbox-а, в котором игроки могут напрямую редактировать все объекты на уровне. Удобство редактирования должно быть близко к Garry's Mod. То есть есть есть можно редактировать любой объект и есть меню спавна объектов.
- Внутриигровой скриптинг для всего (надо выбрать что-то, я склоняюсь к lua)
- ИИ врагов на основе мода Factorio: Rampant

Геймплей:
Шутер с упором на удержание обороны и захват новых областей карты.
Есть как прокачка персонажей так и прокачка пушек. Есть сбор лута как с карты так и с врагов.
Есть строительство баррикад, установка туррелей, ловушек, мин.
Враги многочисленны - от сотен до тысяч (так что важны оптимизации: gpu-instancing, entities, прочее. Большой вопрос - синхронизация по сети. Какие есть подходы к решению этого вопроса)


Основные объекты:
- Статичные объекты
- Навигационные ноды (их будет проще настраивать в с
- Интерактивные объекты
- Спавнеры
- Юниты: Мобы, Герои, прочие юниты



Архитектура игры.
При построении архитектуры ориентируемся на подходы из Unreal Engine (игровые стейты, пешки, абилки) и общую простоту дизайна.
Основные классы архитектуры:
- class GameManager - управляет стейтом игры
- class GameState - стейт игры (синхронизируемый по сет)
- class PlayerState - стейт игрока, внутри GameState может быть много стейтов для разных игроков
- class PlayerController - класс игрока, через него игрок получает доступ к стейту игры и управлению пешками (IPawn)
- class BotController - класс ИИ персонажа, который умеет управлять пешками (IPawn)
- class Director - класс ИИ, управляющий игрой глобально миром
- class Entity - любая игровая сущность. Энтитями являются все объекты внутри игрового мира: местность, декорации, навигация, интерактивные объекты, юниты
- class Unit : Entity, IPawn - активные игровые сущности
  Юниты - это все энтити, которые обладают логикой и действуют активно: герои игроков, союзные и враждебные мобы.
- class CameraOperator - это отдельный класс для управления камерой. Есть префаб по умолчанию, который используется контроллерами игрока, когда они управляют пешками, но отдельные пешки могут иметь свои уникальные варианты камеры.
- class Item : Entity - игровые сущности, которые юниты могут брать с собой
- class Ability - способности юнита, которые он может исполнять (вызываются игроками или ботами) способности могут принадлежать юниту или предмету (последний выдает их тому, кто держит предмет)
- class Map - класс, содержащий в себе список всех объектов на карте. Карты сохраняются не как юнити сцены, а в кастомном формате. Это нужно для того, что бы игроки могли сами собирать карты. При редактировании карты в реалтайме сохраняются по сети.
