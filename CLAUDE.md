# TotalShooter — контекст проекта

Сендбокс-шутер, вид от третьего лица сверху (Alien Shooter / Alien Swarm стиль), кооп/PvP.
Оборона + захват областей карты, прокачка персонажа и оружия, лут, строительство
баррикад/турелей/ловушек/мин. Врагов много — от сотен до тысяч. Полное описание идеи и
геймплея — в `notes.txt`.

Sandbox-режим редактирования в духе Garry's Mod (любой объект можно отредактировать,
меню спавна) + внутриигровой скриптинг. Проект делается автором и друзьями (все —
геймдев-программисты/гейм-дизайнеры) для игры друг с другом, с идеей допиливать игру на
ходу, играя в неё — защита от гриферства/читерства сознательно не в приоритете сейчас.

Подробные архитектурные решения с обоснованиями — [.claude/architecture.md](.claude/architecture.md)
(не грузится автоматически, читать по необходимости). Технические спецификации для реализации
(конкретные поля/методы/потоки) — [.claude/code.md](.claude/code.md). Здесь — только выжимка
решений. Отдельные объёмные темы (например исследование мода Rampant) получают свой файл в
`.claude/` по мере проработки — не тащить их в общий контекст, пока не нужны для конкретной
задачи.

## Архитектура (кратко, без обоснований)

- **Entity** — плоский класс, без иерархии наследования. Композиция через компоненты/флаги
  (например `CanBeControlled`), не подклассы.
- **Массовые юниты** — обычный C# (MonoBehaviour-стиль) по умолчанию; точечный переход
  горячих подсистем (Movement, Targeting/LOS) на `Unity.Jobs`/Burst — только по факту узких
  мест. Централизованные менеджеры вместо `Update()` на каждом объекте, blittable-friendly
  данные в горячих компонентах.
- **Мультиплеер** — player-hosted (listen server) через Unity Netcode for GameObjects +
  Unity Relay.
- **Сетевой статус Entity** — `NetworkObject` только у героев игроков и мета-объектов сессии
  (`GameState`/`PlayerState`/`PlayerController`). Всё остальное (мобы, турели, баррикады,
  лут) — просто `Entity` со своим `EntityId`, без NetworkObject, один общий кастомный канал
  репликации на всё это.
- **Репликация мобов/структур** — command-channel (редкие команды от AI: иди/атакуй/абилка) +
  локальный pathfinding на клиенте + редкая (~1/сек) коррекция позиции от хоста
  (dead reckoning).
- **Урон/killstreak** — host-validated hits (lag compensation), не доверяем самоотчёту
  клиента. Одновременные смертельные удары нескольких игроков → всем assist.
- **Лут** — эксклюзивно первому подобравшему, без дублирования.
- **Скриптинг** — Lua через NLua. Скрипт крутится в `Controller`, `EntityVariables` —
  key-value/blackboard компонент на любом Entity (по умолчанию не сетевой, конкретные ключи
  можно явно продвинуть в репликацию). API скриптам — white-list.
- **World** — карта = папка `Worlds/<world-name>/` (объекты + `Scripts/*.lua` +
  опциональные ассеты), не unity-сцена.
- **Прогресс игрока** — аккаунт (локально у игрока, глобальные метрики) + профиль в мире
  (у хоста, метрики конкретного мира).
- **Права в sandbox** — группы в духе Unix (права объекта в одном `short`), sandbox-режим
  включает хост (для себя и/или других). Правку от не-хоста хост валидирует и применяет сам.
- **Сессия** — `GameManager` (host-only) ведёт лобби/countdown/матч и пишет в реплицируемый
  `GameState`; `NetworkManager` — только транспорт/комната/подключения. `PlayerController`
  скрыт от чужих клиентов, possession-паттерн переключения между пешками (призрак/персонаж).
- **Компоненты** — `Health` (с типами урона), `Spawner` (теги/префабы/кулдаун/автоспавн/лимит)
  как явные переиспользуемые компоненты.
- **Синглплеер** — не отдельный случай, просто хост без клиентов.
